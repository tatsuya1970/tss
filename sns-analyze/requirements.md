# Xトレンド・イベント可視化ツール 要件定義書

## 1. プロジェクト概要
X (旧Twitter) APIを利用し、投稿に含まれる位置情報やテキスト内の地名などを解析することで、特定のエリアで「今、何が盛り上がっているか」（トレンド、イベント、事故、災害情報など）を地図上で可視化するWebベースのツールを開発する。

## 2. 目的・コアバリュー
*   **目的**: 地域ごとの熱量や話題を視覚的に把握できるようにし、お出かけ時の参考情報やマーケティング調査、地域イベントの反響確認などに役立てる。
*   **コアバリュー**:
    *   従来のタイムラインや日本全国のトレンド一覧ではなく、「地図ベース」で直感的に話題の発生源を把握できる。
    *   位置情報を持たない投稿であっても、テキスト解析（地名・施設名抽出）を行うことで、話題の場所を特定しカバレッジを高める。

## 3. ターゲットユーザー
*   **一般ユーザー**: これから遊びに行く場所の盛り上がりや、近隣で起きている出来事（お祭り、事故・遅延、混雑状況など）をリアルタイムに知りたい人。
*   **イベント主催者/マーケター**: イベント会場周辺でのリアルタイムな反響や、特定の地域での局所的なトレンドを知りたい人。
*   **メディア/報道関係者**: 地域単位でのニュースソースや話題のタネをいち早く探している人。

## 4. 機能要件
### 4.1 データ収集機能 (X API連携)
*   **投稿データ取得**: X API v2等を利用し、指定のエリア（バウンディングボックスなど）や特定のキーワードに関する投稿を取得。
*   **位置情報の抽出メカニズム**:
    *   **明示的な位置情報**: 投稿に付与された緯度経度・Place情報（`geo` フィールド）。
    *   **暗黙的な位置情報**: 投稿テキストから地名・施設名・駅名などを抽出し（固有表現抽出）、ジオコーディングAPIで緯度経度に変換。
*   **フィルタリング**: Botによる自動投稿、スパム、位置情報と結びつかないノイズ投稿の除外。

### 4.2 データ分析・集計機能
*   **盛り上がりスコアリング**: 特定エリア内・一定時間枠における投稿数、リポスト数、いいね数から、その場所の「熱量」をスコアリング。
*   **キーワード抽出・クラスタリング**: 同じ場所で同時多発的に呟かれている単語（ハッシュタグ、名詞）を抽出し、何が起きているのかを自律的に要約。
*   **カテゴリ分類**: 話題を「イベント」「グルメ」「交通・事故等のトラブル」「その他」に変更する自動ラベリング機能。

### 4.3 UI/可視化機能（フロントエンド）
*   **地図とヒートマップ表示**: 投稿の密度や盛り上がりスコアを地図上にヒートマップとして可視化（赤いほど熱量が高い）。
*   **ピン・クラスタリング表示**: 話題の中心点にピンを表示し、クリック（タップ）で代表的なキーワードや投稿のサマリーをポップアップ表示。
*   **タイムスライダー機能**: 過去数時間〜数日間の盛り上がりの推移をアニメーションで振り返る機能。
*   **検索・フィルタリング機能**: 「花火」「混雑」などのフリーワードや、時間帯、カテゴリによる絞り込み表示。

## 5. 非機能要件
### 5.1 パフォーマンス・スケーラビリティ
*   **リアルタイム性**: 投稿から集計、マップへの反映までを適度な遅延（例：15分〜30分ごとのバッチ更新）で完了させる。
*   **描画パフォーマンス**: マップ上のデータポイントが数万件に達しても、ブラウザのレンダリングが重くならないよう、適切なデータのクラスタリング（間引き）やベクトルタイルの活用を行う。

### 5.2 X APIの運用制約への対応
*   **レート制限（Rate Limits）対策**: X APIのプラン（Basic / Pro等）に応じた月間取得上限や単位時間あたりのリクエスト数制限に抵触しないよう、データのキャッシュ化と取得頻度の最適化を実装する。
*   **コンプライアンス・利用規約遵守**: X Developer Agreementに基づき、長期間のデータ保存制限への対応（一定期間後のデータパージ）や、再配布に関する規約を守った設計とする。

### 5.3 セキュリティ・プライバシー
*   個人情報保護の観点から、詳細すぎる特定の個人の位置情報を特定できないよう、ヒートマップ表示や適度な範囲での丸め処理（グリッド単位での集計など）を実施する。

## 6. 推奨システム構成・技術スタック
*   **フロントエンド**: 
    *   フレームワーク: React (Next.js) または Vue (Nuxt.js)
    *   地図描画ライブラリ: Mapbox GL JS, MapLibre GL JS, または Leaflet
*   **バックエンド**: 
    *   言語・フレームワーク: Python (FastAPI) または Node.js (Express / NestJS)
    *   ※NLP（自然言語処理）ライブラリを多用するため、Pythonが適している可能性が高い。
*   **データストア**:
    *   データベース: PostgreSQL + PostGIS (空間情報の保存と検索に必須)
    *   キャッシュ: Redis (ヒートマップデータ等の高速読み出し用)
*   **外部API・解析ツール**:
    *   X API v2
    *   Google Maps Geocoding API / Yahoo!ローカルサーチAPI (地名→緯度経度変換)
    *   spaCy / GiNZA 等のNLPライブラリ (固有表現抽出)
*   **インフラ**: AWS (ECS, RDS, ElastiCache) または Google Cloud (Cloud Run, Cloud SQL)
